# Phase 4 – TTS exact recording-path fix

## Basis

This change is applied directly to the user-provided `flowstation-tts.zip` source tree.
No older repository package was used as the base.

## Behaviour

Piper output is first written completely to a private partial file. It is then decoded and materialized into the exact WAV format generated by the local recorder:

- RIFF/WAVE PCM
- signed 16-bit little-endian
- mono
- 8,000 Hz
- canonical 44-byte header

The normalized file is atomically published only after `sync_all()`. Both direct TTS dispatch and dispatch of a prepared preview then call `AudioPlayerHandle::play_recording()` directly. The former `play_generated_audio()` entrypoint has been removed, so no TTS-specific playback branch remains.

After the handoff, source type, full decode, ACELP preparation, CMCE call setup, RF lead-in, playout and release are identical to a recording selected from the dashboard.

## Expected log sequence

```text
TTS: materialized complete recording-format WAV ... format=pcm_s16le/mono/8000Hz
TTS: queueing materialized WAV through exact recording playback entrypoint ...
AudioPlayer: prepared ...
CMCE: starting NEW network call ...
```
